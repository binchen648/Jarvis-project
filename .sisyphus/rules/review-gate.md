# Review Gate Rule

## Rule
本项目的所有工作在**实施**和**审查**两个环节，均自动调用复杂处理专员完成。不再逐一询问用户。

## Workflow

### 实施阶段
1. 判定任务的复杂程度：
   - **复杂**（涉及 2+ 模块、安全敏感逻辑、架构决策、模型/视图/测试综合变更）→ 直接 `task(category="ultrabrain", load_skills=[], run_in_background=false, prompt="...")` 将完整任务描述交给复杂处理专员执行
   - **简单**（单文件 typo、配置字段调整、一行改动）→ 可直接操作，但完成后仍走审查
2. 复杂处理专员返回结果后，验证 output 是否符合预期，然后标记完成

### 审查阶段
1. 每完成一个项目阶段/里程碑后（Phase 交付、大功能块、测试套件补充、关键修复），自动调用复杂处理专员进行审查
2. 审查指令示例：
   ```
   task(
       category="ultrabrain",
       load_skills=[],
       run_in_background=false,
       prompt="全面审查本次变更：[描述范围]。检查：正确性、完整性、性能、安全、项目约定。列出所有问题和修复建议。"
   )
   ```
3. 审查发现的问题 → 全部修复后才标记阶段完成
4. 无需询问用户是否要审查，直接执行

## Scope
- 项目根目录 `D:\Jarvis project\` 下的所有工作
- 包括但不限于：Phase 开发、功能实现、测试补充、重构、配置变更、迁移、部署准备

## 例外
如果用户明确说"不要审查"或"直接做，不用管审查"，则跳过审查步骤。其他情况一律按规则执行。
