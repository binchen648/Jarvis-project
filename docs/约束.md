# Jarvis OS UI Generation Specification V3

You are not designing a dashboard.

You are designing an AI Operating System.

Reference quality level:

* Linear
* Arc Browser
* Cursor
* Raycast
* Vercel
* Gemini

Avoid:

* Traditional admin dashboard
* Bootstrap style cards
* Heavy borders
* Pure black background
* Large tables
* Generic SaaS appearance

Core Design Principles:

1. AI First

The AI is the primary object.
Not the page content.

Every page should feel like it is orbiting around Jarvis.

2. Ambient Intelligence

The interface should feel alive.

Use:

* slow moving gradients
* ambient lighting
* subtle motion
* breathing effects
* glass surfaces

Avoid:

* aggressive animations
* flashy cyberpunk effects
* neon overload

3. Spatial Hierarchy

Every page must follow:

Top Layer:
Jarvis Core Status

Middle Layer:
Primary Work Surface

Right Layer:
Context / Memory / Agent State

Bottom Layer:
Command Layer

4. Visual Language

Background:

Aurora Gradient System

Purple:
#8B5CF6

Blue:
#4F8CFF

Cyan:
#06B6D4

Use blurred radial gradients.

No flat black backgrounds.

Glass Cards:

backdrop blur
soft highlights
minimal borders

5. Motion

Aurora:
30-60 second drift

Core Orb:
8 second breathing animation

Hover:
1-2px lift

Transitions:
200ms-300ms

6. Typography

Large:
32-40px

Section:
20-24px

Body:
14-16px

Meta:
10-12px

Use large spacing.

Avoid dense layouts.

7. AI Presence

Jarvis should always be visible.

Never hide AI functionality behind menus.

Jarvis Core must exist on every page.
