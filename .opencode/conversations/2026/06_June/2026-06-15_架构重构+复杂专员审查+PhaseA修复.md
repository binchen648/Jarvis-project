## 备注

### 踩过的坑
1. **Docker Desktop PID 误杀**：Stop-Process 时未确认 PID 归属，杀了 Docker 导致 PostgreSQL/Redis 下线，连锁导致 channels_redis 阻塞 10 分钟+
2. **com.docker.backend 占用 8000 端口**：即使只启动 db/redis，Docker Desktop 的端口转发引擎也会监听 compose 文件中定义的端口
3. **Windows curl 是 PowerShell 别名**：`curl` 在 PS 中等同于 Invoke-WebRequest，需用 `curl.exe` 才是真正的 curl
4. **PowerShell 管道导致 bash 工具等待**：`| ForEach-Object { Stop-Process }` 的管道链会让 bash 工具认为命令未结束
5. **channels 不在 INSTALLED_APPS**：导致 WebSocket 请求变成 HTTP 404

### 架构决策
- 使用 SQLite demo 配置（config/settings/demo.py）解决 Docker 不稳定性
- 缓存后端改用 LocMemCache 而非 django_redis（demo 环境）
- Channel layer 改用 InMemoryChannelLayer（demo 环境）
- services/ 改名为 infra/agent/ + infra/memory/ + tools/ 保持与 infra/llm/ 同级
- models.py 保留在 infra/llm/ 中（Django 模型容器），业务逻辑迁出
